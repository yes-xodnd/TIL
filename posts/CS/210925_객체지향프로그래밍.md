# 객체지향 프로그래밍

자바스크립트는 멀티 패러다임 언어로 명령형, 함수형, 객체지향의 특징을 모두 가지고 있습니다. 다른 객체지향 언어들과 다르게 클래스가 없고, 프로토타입 기반이라는 특징을 가지고 있습니다. 자바스크립트의 프로토타입 기반 객체지향 언어라는 특성을 이해하기 위해서, 먼저 객체지향 프로그래밍에 대해 정리해보았습니다. 

## 1. 개념

객체지향 프로그래밍은 프로그래밍 패러다임 중 하나입니다.

프로그램을 함수의 집합 또는 명령어들의 집합으로 보는 전통적인 절차지향 프로그래밍 패러다임에서 벗어나, 객체라는 여러 개의 독립된 단위들의 집합으로 이해하고자 하는 관점입니다. 객체지향에서는 객체들 간의 상호작용을 통해 로직을 구성합니다.

## 2. 구성 요소

### 2.1. 클래스

공통된 속성과 메소드를 갖는 객체를 정의한 것으로, 추상화된 데이터 타입이라고 할 수 있습니다. 속성은 객체의 상태, 데이터를 나타내며 메서드는 객체가 실행해야 할 연산을 의미합니다.

### 2.2. 객체 

실제로 메모리에 할당된 클래스의 인스턴스로, 각각 고유의 속성을 가지며 메소드를 수행할 수 있습니다. 같은 클래스로 생성된 객체들은 메소드의 정의를 공유하여 메모리를 경제적으로 사용할 수 있습니다.

### 2.3. 메시지

객체들 간 상호작용을 위한 수단으로, 객체에서 객체로 전달됩니다.



## 3. OOP의 특징

객체지향 프로그래밍 언어는 일반적으로 다음과 같은 특징을 가지고 있습니다.

### 3.1. 캡슐화

데이터 구조와 데이터를 다루는 방법을 하나로 묶는 것을 의미합니다. 관련된 데이터와 함수를 묶어 응집도를 높이면서, 정보 은닉을 통한 인터페이스 단순화로 결합도를 낮출 수 있습니다.

### 3.2. 정보 은닉

다른 객체에게 자신의 정보를 숨기고 자신의 연산을 통해서만 접근할 수 있게 하는 것을 의미합니다. 객체의 수정으로 발생하는 부작용(side effect)를 줄여, 유지보수와 확장을 용이하게 합니다.

### 3.3. 상속

상위 개념의 특징을 하위 개념이 물려받는 것을 의미합니다. 상속은 캡슐화를 유지, 클래스의 재사용을 용이하게 합니다. 상속 후 기존의 속성, 연산을 수정하거나 새로운 속성, 연산을 추가하여 클래스를 확장해 사용할 수 있습니다.

### 3.4. 추상화

주어진 문제, 시스템에서 중요하고 관계있는 부분만 중점을 두고 모델링 하는 것을 의미합니다. 데이터의 추상화를 통해 정된 데이터 타입을 추상 데이터 타입이라고 합니다. 일반적으로 객체지향 프로그래밍에서 추상 데이터 타입을 클래스, 추상 데이터 타입의 인스턴스를 객체, 정의된 연산을 메소드, 메소드의 호출을 생성자라고 합니다. 

### 3.5. 다형성

하나의 클래스나 메소드가 다양한 방식으로 동작이 가능한 것을 의미합니다. 구체적으로는 오버라이딩, 오버로딩을 통해 구현됩니다.

- 오버로딩 : 같은 메서드가 인수의 수나 타입에 따라서 다르게 작동하게 하는 것
- 오버라이딩 : 상속받은 메서드를 재정의하여 사용하는 것



## 4. SOLID 원칙

로버트 마틴이 제안한 객체지향 프로그래밍 및 설계의 기본적인 원칙으로, 각 원칙의 첫 글자를 따서 SOLID 원칙이라고 부릅니다.

### 4.1. 단일 책임 원칙 Single Responsibility Principle

하나의 클래스는 하나의 책임만 지녀야 한다.

### 4.2. 개방-폐쇄 원칙 Open/Closed Principle

소프트웨어 요소는 확장에는 열려있으나 변경에는 닫혀 있어야 한다.

자식 클래스에서 기능 추가/수정할 수 있으나
자식 클래스를 위해 부모 클래스를 수정할 필요 없음

### 4.3.리스코프 치환 원칙 Liskov's Substitution Principle

자식 클래스는 언제나 자신의 부모 클래스를 교체할 수 있다.

### 4.4. 인터페이스 분리 원칙 Interface Segregation Principle

자신과 관련 없는 인터페이스는 구현하지 않아야 한다.

### 4.5. 의존성 역전 법칙 Dependency Inversion Principle

고차원 클래스가 저차원 클래스에 의존하면 안된다.
